# プロシージャル建物生成ツール仕様書（Unity + ProBuilder）

## 1. 目的

Unity上で建物を効率的に作成するための **半自動プロシージャル建物生成ツール** を開発する。
ユーザーは **ProBuilder を用いて建物の大まかな形状を作成**し、その形状に対してツールが **建築的意味づけ（Semantic）を推定**する。
ユーザーは必要に応じて意味づけを修正し、最終的に **壁・屋根・柱・アーチなどのモジュールを自動配置**する。

本ツールは

* 手作業による建物制作の効率化
* プロシージャル生成による量産
* 手動編集との共存

を目的とする。

---

# 2. 基本設計方針

## 2.1 半自動生成

生成は **完全自動ではなく半自動**とする。

ツールは形状から建築的意味を推定するが、
最終判断はユーザーが修正できる。

理由:

* 建築構造は形状だけでは判断できない場合が多い
* ユーザーの意図（入口・装飾など）を反映する必要がある

---

## 2.2 編集可能性の確保

生成後も以下が可能であることを前提とする。

* 個別パーツ差し替え
* 部分再生成
* 手動配置
* 自動生成の再実行

生成結果は単なるメッシュではなく
**意味情報を保持した生成データ構造**として扱う。

---

## 2.3 フェーズベースワークフロー

建物生成は以下のフェーズで進行する。

```
Shape → Semantic → Generate → PostEdit → Bake
```

これにより責務を明確に分離する。

---

# 3. フェーズ構成

---

# Phase 1: Shape（形状作成）

## 概要

ユーザーが **ProBuilder を用いて建物のラフ形状を作成する段階**。

この段階ではまだ建築的意味は存在しない。

### 主な用途

* 建物ボリューム作成
* 大まかなシルエット決定
* フロア構成検討

### 使用ツール

* ProBuilder
* Unity Scene編集

### 形状例

* 直方体
* 三角柱
* 複数ボリュームの組み合わせ

---

# Phase 2: Semantic（意味づけ）

## 概要

Shape を解析し、
**建築的意味（Semantic）を自動推定する段階。**

ユーザーはこの結果を確認し、
必要に応じて手動修正できる。

---

## 2.1 自動分類

面の法線などを用いて自動推定する。

例:

| 条件                  | 分類      |
| ------------------- | ------- |
| normal.y > 0.9      | Floor   |
| normal.y < -0.9     | Ceiling |
| abs(normal.y) < 0.2 | Wall    |

追加条件で

* Roof
* OuterWall
* InnerWall

などに分類可能。

---

## 2.2 Semantic可視化

Semanticは **色分け表示**される。

例:

| Semantic    | Color  |
| ----------- | ------ |
| Wall        | Red    |
| Floor       | Green  |
| Roof        | Blue   |
| Ceiling     | Cyan   |
| OpeningHost | Yellow |

これによりユーザーは一目で意味づけを確認できる。

---

## 2.3 手動修正

ユーザーは以下を変更可能。

* 面の役割
* 開口面指定
* 特殊壁
* 屋根指定

例:

```
Right Click → Set Face Role → Entrance
```

---

# 4. Generate（建物生成）

Semantic情報に基づき
**モジュールアセットを配置する。**

---

## 4.1 モジュール種類

### 壁

* Wall Panel
* Window Wall
* Door Wall

### 屋根

* Flat Roof
* Gable Roof
* Roof Edge

### 床

* Floor Tile
* Ceiling Tile

### 構造

* Pillar
* Beam

### 装飾

* Arch
* Balcony
* Frame

---

## 4.2 モジュール配置ルール

### 壁

壁面をグリッド分割して配置。

```
wall face
↓
grid split
↓
panel placement
```

---

### 柱

柱は **自動認識ではなくルール生成**。

例:

* コーナー
* 壁分割境界
* 一定間隔

---

### アーチ

アーチは **手動指定の開口に対して生成**。

---

# 5. Post Edit（生成後編集）

生成後にユーザーが直接編集可能。

可能操作:

* 個別パーツ差し替え
* 削除
* 手動追加
* Semantic変更→再生成

---

# 6. Bake / Export（最終化）

生成結果を最終データへ変換する。

---

## 6.1 出力モード

ユーザーは以下から選択可能。

### Separated

パーツごとにGameObjectを保持

用途:

* 編集
* デバッグ

---

### Grouped

カテゴリごとにまとめる

例

```
Walls
Roofs
Floors
Columns
```

用途:

* 管理しやすい
* 適度なカリング

---

### Combined

可能な範囲でメッシュ結合

用途:

* 描画負荷削減
* 最終アセット化

---

# 7. 追加機能

---

# A. 自動分類の信頼度表示

自動分類結果に **Confidence（信頼度）** を持たせる。

例

| 状態  | 意味        |
| --- | --------- |
| 確定  | アルゴリズムで明確 |
| 要確認 | 曖昧        |

UI例

```
Green = confident
Orange = uncertain
```

ユーザーが修正しやすくなる。

---

# B. 再生成範囲指定（重要）

全体再生成だけでなく
**部分再生成**を可能にする。

再生成範囲:

* 全体
* 面単位
* 階単位
* 屋根のみ
* 壁のみ

例

```
Rebuild → Selected Face
```

これにより作業効率が向上する。

---

# C. 手編集保護（重要）

ユーザーが手動編集したパーツを
再生成で破壊しないようにする。

各パーツに

```
Lock
```

フラグを持たせる。

Lockされたパーツは

* 再生成対象外
* 自動更新無効

となる。

# まとめ

本ツールは

* **ProBuilderによる形状作成**
* **Semanticによる意味づけ**
* **モジュール配置による建物生成**

の3段階構造を採用する。

生成は **半自動**とし、
ユーザーによる修正と編集を前提とした設計とする。

これにより

* プロシージャル生成の効率
* 手作業の柔軟性
* 実用的なワークフロー

を両立する。
