## ⚠️ レンダーパイプラインと動作状況

### 実際に動作するシェーダー

| シェーダー | 書き方 | HDRP で動くか | 備考 |
| --- | --- | --- | --- |
| `URP_Hologram.shader` | `CGPROGRAM` + `UnityCG.cginc` | **✅ 動作する** | **← こちらを使うこと** |
| `HDRP_Hologram.shader` | `HLSLPROGRAM` + `ForwardOnly` | ❌ 描画されない | 参考実装として残置 |

### HDRP_Hologram.shader が動かない理由

HDRP ネイティブの `HLSLPROGRAM` + `"LightMode" = "ForwardOnly"` による実装は、
HDRP の完全なシェーダーフレームワーク（ShaderPass / VertMesh / FragForward 等）を
組み込まないと HDRP のドローコールに含まれない。

一方、**`CGPROGRAM`（レガシー CG）はサブシェーダータグに RenderPipeline 指定がない場合、
HDRP でもフォールバックとして描画される**。`URP_Hologram.shader` はこの仕組みを利用しており、
HDRP / URP / Built-in パイプラインすべてで動作する。

### 使用ファイルまとめ

```
Assets/_Project/Shaders/Hologram/
├── HologramCore.hlsl       # 共通関数ライブラリ (HLSL版: HDRP_Hologram.shader から参照)
├── HDRP_Hologram.shader    # ❌ HDRP で描画されない。参考実装として残置
└── URP_Hologram.shader     # ✅ 実際に使うシェーダー (HDRP 含む全パイプラインで動作)
```

## 実装した3つの方法

### 方法A — Fresnel 発光

視線方向と法線の内積を使い、エッジ部分を強く発光させる。

```hlsl
// HologramCore.hlsl より
float vDotN  = saturate(dot(viewDir, normalWS));
float fresnel = pow(1.0 - vDotN, power);
```

| パラメータ | 説明 | 推奨値 |
|---|---|---|
| `Fresnel Power` | エッジの鋭さ。大きいほどエッジのみ光る | 2〜5 |
| `Fresnel Intensity` | 発光の強さ倍率 | 1〜3 |

---

### 方法B — スキャンライン

UV.y 座標に sin を適用し、時間でスクロールする走査線を生成する。

```hlsl
float raw     = sin(uvY * frequency + time * speed);
float scanline = raw * 0.5 + 0.5;  // [0,1] に正規化
```

| パラメータ | 説明 | 推奨値 |
|---|---|---|
| `Scanline Frequency` | 走査線の密度（高いほど細かい） | 40〜150 |
| `Scanline Speed` | スクロール速度 | 1〜5 |
| `Scanline Intensity` | 明暗の深さ。0 で無効 | 0.3〜0.7 |

---

### 方法C — グリッチ（ノイズ変位）

バリューノイズを使い、頂点変位（形状のズレ）と色ずれを発生させる。
ランダムなタイミングで発火するゲート処理付き。

```hlsl
// 頂点変位 (vertex shader)
float gate  = step(0.95, Hologram_Noise(float2(time * speed * 0.31, 7.3)));
float3 offset = float3(noise * intensity * gate, 0, 0);

// 色ずれ (fragment shader)
float3 glitchColor = float3(noise * gate * intensity * 2.0, 0, 0);
```

| パラメータ | 説明 | 推奨値 |
|---|---|---|
| `Glitch Intensity` | 変位量と色ずれ量 | 0.02〜0.1 |
| `Glitch Speed` | グリッチの変化速度 | 5〜15 |

## クイックスタート

### HDRP での使用手順

1. `Assets/_Project/Material/Hologram/` フォルダを作成
2. **右クリック → Create → Material** で新規マテリアルを作成
3. マテリアルの Shader を `Custom/HDRP/Hologram` に変更
4. `Hologram Color` に HDR シアン系色を設定
   例: `(0, 0.8, 1, 1)` に HDR 強度 2〜3
5. 適用したいメッシュの Mesh Renderer にマテリアルをアサイン

> **注意**: HDRP でホログラムを自然に見せるには
> `Volume → Bloom` を有効にするのが効果的（`_HologramColor` を HDR にすること）

### Bloom の推奨設定（HDRPボリューム）

```
Bloom:
  Intensity: 0.5 〜 1.5
  Threshold: 1.0
  Scatter: 0.7
```

## HDRP と URP の違い

| 項目 | HDRP | URP |
|---|---|---|
| RenderPipeline タグ | `HDRenderPipeline` | `UniversalPipeline` |
| LightMode タグ | `ForwardOnly` | `UniversalForward` |
| カメラ位置取得 | `GetCurrentViewPosition()` | `_WorldSpaceCameraPos.xyz` |
| コアインクルード | `HDRP/.../ShaderVariables.hlsl` | `URP/.../Core.hlsl` |
| `HologramCore.hlsl` | 共通使用 ✓ | 共通使用 ✓ |

## Shader Graph レシピ

HDRP / URP 共に **Unlit Graph** をベースに以下のノード構成で再現できる。

### 準備

1. **右クリック → Create → Shader Graph → HDRP Unlit Graph**（または URP Unlit Graph）
2. Graph Inspector → Surface Type: **Transparent**
3. Graph Inspector → Blending Mode: **Alpha**

---

### ノード構成図（テキスト表現）

```
[Time] ─────────────────────────────────────────────────────────┐
                                                                  │
[Position (World)] ──[Normalize]──────────────────────────────┐  │
[Normal Vector (World)] ──[Normalize]──────────────────────┐  │  │
                                                            │  │  │
                                          ┌─────────────────┘  │  │
                                          │   ─[Dot]────────────┘  │
                                          │       │                 │
                                          │   ─[One Minus]         │
                                          │       │                 │
                  [FresnelPower] ──────────────[Power] = fresnel    │
                                                │                  │
                  [FresnelIntensity] ─────[Multiply]               │
                                                │                  │
              ─────────────────────────────[Add(1+)] = fresnelTerm │
                                                                   │
[UV.y] ──[Multiply(freq)] ──[Add(time*speed)] ──[Sine] ──[Remap]  │
                              ↑                           │        │
                          [Time]──[Multiply(speed)]       │        │
                                                      scanlineTerm │
                                                                   │
[UV.y,Time] ──[Noise]──[Step(0.97)]──[Multiply] = glitchColor     │
                                                                   │
[HologramColor] ──[Multiply(fresnelTerm)] ──[Multiply(scanline)] ──[Add(glitch)] → Base Color
                                                                    │
[Alpha] ──[Multiply(0.4 + fresnel*0.6)] ──[Multiply(scanline)] ─── → Alpha
```

---

### ノード詳細手順

#### Step 1: Fresnel ノード

```
方法A の実装

1. [Normal Vector] ノード追加 (Space: World)
2. [View Direction] ノード追加 (Space: World)
3. [Dot Product] ノード: Normal × ViewDir を接続
4. [One Minus] ノード: Dot の出力を接続
5. [Power] ノード:
     Base  ← One Minus の出力
     Exp   ← Float プロパティ "FresnelPower" (デフォルト 3.0)
6. [Multiply] ノード:
     A ← Power の出力
     B ← Float プロパティ "FresnelIntensity" (デフォルト 2.0)
   → 出力: fresnel_scaled
7. [Add] ノード: fresnel_scaled + 1.0 → fresnelTerm
```

#### Step 2: スキャンライン ノード

```
方法B の実装

1. [UV] ノード → Y チャンネルを [Split] で取り出す
2. [Multiply]: UV.y × Float "ScanlineFrequency" (80)
3. [Time] ノード → [Multiply] Float "ScanlineSpeed" (3.0)
4. [Add]: 上記2つを合算 → sin の位相
5. [Sine]: Add の出力を接続
6. [Remap]: -1〜1 → 0〜1 (または [Add(1)][Multiply(0.5)])
7. [Lerp]:
     A ← 1.0
     B ← Sine の結果
     T ← Float "ScanlineIntensity" (0.5)
   → 出力: scanlineTerm
```

#### Step 3: グリッチ ノード

```
方法C の実装 (色ずれのみ、頂点変位は Shader Graph では Custom Function ノードが必要)

1. [UV] → Y チャンネルを取り出す
2. [Combine] で float2(UV.y * 5, Time * GlitchSpeed) を作る
3. [Simple Noise] ノード: 上記 float2 を UV に接続
4. [Time] → GlitchSpeed で Multiply した値を別の [Simple Noise] へ
5. [Step]: Threshold=0.97 でゲートを作る
6. [Multiply]: Noise × Gate × GlitchIntensity × 2
7. [Combine] で R チャンネルのみに値を入れる → glitchColor (float3)
```

#### Step 4: 合成・出力

```
1. [Color] プロパティ "HologramColor" (HDR有効)
2. [Multiply]: HologramColor × fresnelTerm
3. [Multiply]: 上記 × scanlineTerm
4. [Add]: 上記 + glitchColor → Base Color 出力へ

5. [Float] プロパティ "Alpha" (0.75)
6. [Add]: fresnel_scaled × 0.6 + 0.4
7. [Multiply]: 上記 × Alpha × scanlineTerm
8. [Saturate] → Alpha 出力へ
```

#### Step 5: 頂点変位 (Custom Function ノード)

頂点グリッチをShader Graphで実装するには **Custom Function** ノードが必要。

```hlsl
// HologramGlitch_Node.hlsl として保存して Custom Function にリンク
void HologramGlitch_float(float uvY, float intensity, float speed, float time,
                          out float3 offset)
{
    // ここに HologramCore.hlsl の Hologram_GlitchOffset と同じ処理を記述
    float hash = frac(sin(dot(float2(uvY * 10.0, time * speed),
                             float2(127.1, 311.7))) * 43758.5453);
    float gate  = step(0.95, frac(sin(dot(float2(time * speed * 0.31, 7.3),
                                         float2(127.1, 311.7))) * 43758.5453));
    offset = float3(hash * intensity * gate, 0.0, 0.0);
}
```

Custom Function ノード設定:
- Source: **File** → 上記ファイルをリンク
- Function Name: `HologramGlitch_float`
- 入力: `uvY (Vector1)`, `intensity (Vector1)`, `speed (Vector1)`, `time (Vector1)`
- 出力: `offset (Vector3)`
- [Position] ノードの出力に加算して **Vertex Position** に接続

## トラブルシューティング

| 問題 | 原因 | 対処 |
|---|---|---|
| 真っ黒になる | Bloom 無効 / HDR 未設定 | Volume に Bloom 追加、Hologram Color を HDR に |
| 透明にならない | ZWrite が On になっている | マテリアルの ZWrite を Off に確認 |
| スキャンラインが見えない | Frequency が低い | 値を 50 以上に上げる |
| グリッチが激しすぎる | Intensity が大きい | 0.02〜0.05 程度に下げる |
| URP シェーダーがコンパイルエラー | URP 未インストール | Package Manager で URP を追加 |
| 法線が反転して見える | Cull の問題 | `Cull Off` に変更（両面表示） |

## 技術メモ

### なぜ Fresnel でエッジが光るのか

`dot(viewDir, normal)` は視線と法線が平行（正面）なら **1**、垂直（エッジ）なら **0** になる。
`1 - dot(...)` で反転し、`pow()` で急峻にすることでエッジのみ強く光る。

### なぜ `_Time.y` を使うか

`_Time` は Unity が自動設定するシェーダー定数:
- `_Time.x` = time / 20
- `_Time.y` = time (秒)  ← 使用
- `_Time.z` = time × 2
- `_Time.w` = time × 3

### Bloom との連携

`_HologramColor` を HDR（輝度 > 1）にすることで、Post Processing の Bloom が自動的にグロー効果を付加する。
シェーダー側は発光色を出力するだけで良く、Bloom は HDRP/URP の Volume で制御できる。
